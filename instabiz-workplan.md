# InstaBiz Development & Training Work Plan

## Project Overview
**Project Name:** InstaBiz  
**Training Platform:** InstaSkul.com  
**Tech Stack:** Next.js (App Router), Prisma, PostgreSQL  
**Duration:** 12-16 weeks

---

## Dual-Objective Strategy

### Core Principle
Every development phase will be executed with **dual documentation**:
1. **Development Execution** - Building InstaBiz features
2. **Training Content Creation** - Recording lessons, tutorials, and best practices for InstaSkul.com

### Documentation Format Per Phase
- **Video Tutorials** - Screen recordings with voice-over explanations
- **Written Guides** - Step-by-step markdown documentation
- **Code Repositories** - Staged commits showing progression
- **Challenge Exercises** - Practice problems for students
- **Quiz Questions** - Assessment materials

## Parallel Workflow Strategy

### Daily Development Rhythm
**Morning (3-4 hours):** Development work
- Write code
- Implement features
- Make git commits with detailed messages

**Afternoon (2-3 hours):** Documentation
- Record screen while explaining what was built
- Write detailed guides
- Create diagrams and visuals
- Prepare code examples

**End of Week:** Content Production
- Edit videos
- Finalize written content
- Create exercises and quizzes
- Publish to InstaSkul.com

### Weekly Cycle
- **Monday-Wednesday:** Feature development + daily documentation
- **Thursday:** Complete feature, write comprehensive guide
- **Friday:** Record tutorial video, create exercises
- **Weekend:** Edit content, publish to InstaSkul.com

## Tools and Setup for Content Creation

- **Screen Recording:** OBS Studio / Loom

- **Thumbnails:** Figma / Canva

- **Written Content:** Markdown with code blocks
- **Diagrams:** Excalidraw / Mermaid.js
- **Screenshots:** Annotated with arrows/highlights
- **Code Hosting:** GitHub with detailed READMEs

### Code Management
- **Repository Structure:**
  ```
  instabiz/
  ├── main (production)
  ├── develop (integration)
  └── feature/* (per module branches)
  ```
- **Commit Strategy:** One feature per commit with detailed messages
- **Tags:** Tag each phase completion for students to clone

## Risk Management

### Development Risks
- **Risk:** Feature creep extending timeline
- **Mitigation:** Strict MVP scope, move nice-to-haves to Phase 9

- **Risk:** Technical blockers (API issues, etc.)
- **Mitigation:** Document troubleshooting as bonus content

### Content Creation Risks
- **Risk:** Falling behind on documentation
- **Mitigation:** No feature complete without corresponding content

- **Risk:** Content quality inconsistency
- **Mitigation:** Content review checklist before publishing
